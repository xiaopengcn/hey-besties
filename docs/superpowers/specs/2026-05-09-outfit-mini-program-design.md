# 今日灵感穿搭小程序设计说明

## 1. 产品定位

这是一个给年轻女生在出门前快速获取穿搭灵感的微信小程序。用户先选择出行情景，例如约会、闺蜜出游、看展、逛街、下午茶，系统再生成一套“今日主题 + 三个关键元素”，并通过纸片人模特同步展示一套模板化穿搭结果。

首版重点不是精准推荐商品，也不是完整衣橱管理，而是做到三件事：

- 让用户在 10 秒内得到一个明确、好玩、可执行的穿搭方向
- 让结果看起来足够懂场景，不像随机抽签
- 让页面和纸片人结果适合截图分享

## 2. 目标用户

- 年轻女生
- 高频日常场景：约会、闺蜜出行、看展、逛街、下午茶
- 使用时机：出门前、换衣服前、和朋友讨论穿搭时

## 3. 首版范围

### 包含

- 场景选择生成
- 今日主题生成
- 三个穿搭元素生成
- 纸片人模特模板换装
- 结果重抽
- 本地收藏喜欢的结果
- 适合微信生态的分享文案和分享海报页面

### 不包含

- 用户登录与云端账号
- 上传真人照片换装
- AI 生图
- 电商下单
- 真实衣橱录入

## 4. 核心体验

### 首页

首页直接展示一句有情绪的引导文案和场景卡片。用户选中场景后点击主按钮生成，不需要填写复杂信息。

### 结果页

结果页需要同时给出三层结果：

- 场景理解：今天是什么局
- 文案灵感：今日穿搭主题
- 可视化结果：纸片人模特换装预览

页面按钮聚焦 3 个动作：

- 换一套
- 收藏这一套
- 生成分享图

### 收藏页

收藏页保存用户喜欢的结果，用于回看，不做复杂管理。

## 5. 生成逻辑

生成不是纯随机，而是“场景驱动的有限随机”。

每个场景关联一组：

- 主题池
- 色彩池
- 材质池
- 风格元素池
- 服装模板池

例如“闺蜜出行”更偏轻快、俏皮、好拍照；“约会”更偏柔和、精致、带一点记忆点。生成时先选主题，再补三类元素，最后根据主题和元素组装出一套穿搭模板。

## 6. 模特模板换装方案

首版采用纸片人插画风，不做真人感。核心原因是：

- 风格统一
- 素材成本低
- 审核和渲染风险小
- 更适合模板叠层实现

### 模特结构

纸片人由固定身体和可替换服饰层组成：

- 头部与发型：首版固定
- 脸部：首版固定
- 身体底图：固定
- 上装层：可替换
- 下装层：可替换
- 连衣裙层：可替换
- 外搭层：可选
- 配饰层：可选

### 服饰模板分类

首版只做少量基础模板，但保证组合成立：

- 上装：衬衫、短袖、针织上衣
- 下装：长裤、短裤、长裙、短裙
- 连衣裙：短裙款、长裙款
- 配饰：发卡、包包、项链

### 组合规则

- 如果选中连衣裙，则不上下装同时出现
- 如果是上装 + 下装路线，则按场景选择合适的下装
- 配饰根据风格元素决定是否附加
- 颜色和材质通过预定义 token 影响服装层样式

### 材质表达

首版不做真实纹理贴图，而是用模板化视觉表达：

- 麻料：浅颗粒线条、低饱和、偏自然
- 牛仔：蓝调、明线、硬挺边线
- 棉感：纯色、柔和阴影
- 漫画感：高对比描边与跳色细节

## 7. 视觉设计方向

整体气质要年轻、有活力、带一点时尚手账感，避免通用企业蓝或普通工具感。

### 视觉关键词

- 元气
- 轻时尚
- 贴纸感
- 糖果色
- 漫画排版
- 柔和渐变

### 色彩方向

- 主色：珊瑚粉 / 蜜桃橙
- 辅色：奶油黄 / 薄荷绿 / 天空蓝
- 中性色：奶白 / 雾灰 / 深咖

### 页面层次

- 背景不是纯色，使用大面积渐变和柔和形状
- 卡片使用圆角和轻投影
- 标签使用高识别配色
- 按钮强调大而明确，适合单手操作

## 8. 小程序布局与状态栏策略

为了适配微信小程序不同机型，首版统一采用自定义导航栏。

### 约束

- 读取 `wx.getWindowInfo()` 与 `wx.getMenuButtonBoundingClientRect()`
- 计算 `statusBarHeight`、`navBarHeight`、`capsule` 占位
- 页面顶部统一通过 CSS token 和公共数据注入

### 目标

- 首页和结果页都能自然延展到顶部
- 标题、返回按钮、胶囊区对齐稳定
- iPhone 刘海屏与安卓机型都不挤压内容

## 9. 技术设计

首版采用微信原生小程序，不引入重型框架。

### 原因

- 交付快
- 结构轻
- 与原生页面和分享能力贴合
- 更适合首版快速验证

### 技术组成

- 视图层：WXML + WXSS
- 逻辑层：JavaScript
- 本地存储：`wx.setStorageSync`
- 核心规则：本地数据驱动生成器
- 分享图：先做页面化海报，不依赖服务端

## 10. 页面结构

### pages/home

- 顶部 hero 区
- 场景选择区
- 今日灵感引导文案
- 生成按钮

### pages/result

- 自定义导航栏
- 今日主题卡
- 元素标签区
- 纸片人模特换装舞台
- 操作按钮区

### pages/favorites

- 收藏列表
- 空状态插画文案

### pages/poster

- 分享海报预览
- 主题与标签展示
- 模特静态展示

## 11. 成功标准

如果首版达成以下结果，就说明方向成立：

- 用户能在 3 次点击内看到第一套结果
- 不同场景生成结果明显不同
- 模特换装不会出现不合理叠穿
- 收藏和分享动作顺畅
- 页面视觉在小程序里看起来像产品，不像 demo
