# 今日灵感穿搭小程序二阶段设计说明

## 1. 目标

二阶段是在现有首版玩法不变的前提下，把产品从“可用 demo”提升到“可传播、可分享、视觉更完整”的程度。

本阶段只做三件事：

- 把分享图从预览页升级成真正可导出的图片
- 扩充纸片人服饰模板，让结果更丰富、更耐玩
- 提升颜色和材质表现，让穿搭看起来不只是纯色块

## 2. 范围

### 包含

- 使用小程序 canvas 真正生成海报图片
- 保存海报到相册
- 新增更多服饰模板
- 新增更多颜色组与风格标签
- 把材质表现从基础 CSS 升级成“CSS + 贴图/装饰层”混合模式
- 用 imagegen 生成少量统一风格素材，服务于海报和服饰质感

### 不包含

- 真人上传试衣
- 后端图片服务
- 云端海报缓存
- 电商商品链接

## 3. 海报导出方案

### 目标体验

用户在结果页点击“生成分享图”后，进入海报页并看到：

- 一张版式完整的分享海报
- 一个“保存到相册”按钮
- 一个“重新生成”或“返回修改”入口

### 渲染方式

采用小程序 canvas 分层绘制，不依赖截图。

绘制内容包括：

- 柔和渐变背景
- 贴纸感装饰素材
- 场景标题
- 穿搭主题
- 三个元素标签
- 纸片人模特静态版
- 底部分享文案

### 架构拆分

- `utils/poster-model.js`
  负责把 outfit 数据转换成可绘制的海报模型
- `utils/poster-renderer.js`
  负责 canvas 绘制、导出临时文件、保存图片
- `pages/poster/*`
  负责展示预览、触发导出、处理权限与状态

### 原则

- 海报布局与导出逻辑分离
- 页面展示和图片导出尽量共用同一份海报模型
- 导出失败时给明确提示，不让用户误以为已经保存

## 4. 模板扩充方案

首版服饰数量偏少，二阶段将服饰结构升级为“品类 + 版型 + 细节风格”。

### 新增上装

- 泡泡袖上衣
- 短开衫
- 背心叠穿
- 印花短 Tee

### 新增下装

- 百褶短裙
- A 字短裙
- 牛仔半裙
- 阔腿西裤

### 新增连衣裙

- 吊带连衣裙
- 茶歇裙
- 衬衫裙

### 新增可选层

- 薄开衫
- 牛仔外搭

### 新增配饰

- 腋下包
- 棒球帽
- 爱心耳饰
- 漫画徽章

## 5. 组合规则升级

为了避免“模板多了以后越搭越乱”，组合规则需要更细：

- 场景先决定主路线：连衣裙 / 上下装
- 主题和 vibe 决定廓形倾向：甜感、利落、轻松、文艺
- 材质优先筛选可搭服装
- 配饰按主题和 accent 关联，不是完全随机
- 外搭只在部分风格和场景下出现

## 6. 配色和材质升级

### 配色策略

从“一个颜色名”升级为“颜色组”：

- 主色
- 辅色
- 高光色
- 深线稿色

例如“蜜桃粉”不只是一个粉色，而是一组适合服饰高光、边缘、阴影的粉色层次。

### 材质策略

从纯 CSS 线条升级为两层：

- 基础形体：CSS 绘制服饰轮廓
- 质感层：通过重复纹理、渐变、装饰图片增强材质识别

### 重点材质

- 麻料：浅颗粒、自然纤维感
- 牛仔：斜纹、拼线、口袋线
- 针织：细密重复纹
- 雪纺：轻透叠层
- 缎面：高光偏移

## 7. imagegen 素材策略

imagegen 不用来生成整个人物，而是生成更可控的小面积位图素材。

### 适合用 imagegen 生成的内容

- 贴纸风小装饰
- 漫画星星、爱心、闪光点缀
- 细颗粒布料纹理
- 牛仔和针织风格化小纹理
- 海报角落装饰元素

### 不适合直接用 imagegen 的内容

- 整体纸片人身体
- 主体服装结构
- 依赖精确可控边界的轮廓图层

### 原因

纸片人主体轮廓需要精确、可叠层、可维护，用代码和模板表达更稳定。imagegen 更适合做“视觉质感增强层”。

## 8. 页面变化

### 结果页

- 增加更丰富的服装细节
- 若存在外搭，则展示额外层
- “生成分享图”按钮文案保持不变

### 海报页

- 新增 canvas 节点
- 新增“保存到相册”主按钮
- 显示导出中状态

## 9. 技术边界

- 保持微信原生小程序方案
- 逻辑测试仍使用 Node 内建测试
- 画布绘制逻辑单独模块化，避免堆进页面 js
- 服饰数据保持本地配置驱动，便于后续继续扩展

## 10. 成功标准

如果二阶段完成后达到以下结果，就说明升级成功：

- 用户能真正保存一张完整海报到相册
- 同一场景连续生成几次，服饰组合明显更多样
- 牛仔、针织、麻料等材质一眼能分辨
- 海报在视觉上像一个可以发给朋友的成品，而不是页面截图
